# 自动获客功能设计文档

## 2026-07-08 视觉重设计定稿

### 业务理解

自动获客不是“输入一个官网再解析”，而是销售已经有目标画像，但还没有具体客户名单。页面必须帮助销售从产品、国家、行业、客户类型、公开数据源和平台入口出发，形成可执行的搜客任务，最后把候选客户推进到客户、商机、待办和导出表。

### Figma版式稿说明

本会话已读取 Figma 技能说明，但当前工具列表没有暴露可写入 Figma 的 `use_figma` MCP 工具，因此本次先按 Figma 设计流程输出页面级定稿，并同步落地到代码。等 Figma MCP 写入工具恢复后，可用同一结构在画布中重建：

- 画布尺寸：1440 x 1080，CRM 工作台密度。
- 页面结构：顶部任务指挥条、左侧创建任务、右侧任务队列、下方结果库、右侧窄栏画像与动作。
- 色彩策略：白底、深墨文字、蓝绿作为状态强调，少量琥珀提示，不使用大面积彩色条。
- 字号策略：标题 22-24，分区标题 15-16，表格和表单 12-13，按钮 12，整体更密。
- 圆角策略：卡片 8px，输入框 6px，按钮 6px，符合后台工具质感。

### 三轮自我辩论与修改

第一轮：入口是“官网输入”还是“获客条件”？

- 方案 A：保留官网输入作为主入口，开发成本低。
- 问题：这与自动获客的真实含义冲突，用户没有官网时才需要搜客。
- 定稿：官网/平台链接导入降级为辅助入口。主入口改为产品、市场、行业、客户类型、渠道和数量。

第二轮：页面做成“结果表优先”还是“任务优先”？

- 方案 A：结果表优先，直接看到数据。
- 问题：没有任务过程，用户不知道系统从哪里找、跑到哪一步、哪些来源是真接入。
- 定稿：第一屏左侧创建任务，右侧任务队列，底部结果库。任务队列展示进度、渠道、阶段和下一步动作。

第三轮：视觉是模仿参考图还是建立 GoodJob 自己的风格？

- 方案 A：照参考图做蓝绿进度条和大圆环。
- 问题：过于接近，且圆环占空间，第一屏密度不足。
- 定稿：保留“任务卡 + 阶段 + 渠道 + 进度”的信息结构，改成 GoodJob 的精致商务风：紧凑状态芯片、小型进度轨、右侧任务得分，不照搬参考图。

第四轮：美工验收

- 信息层级：通过。用户第一眼能看到“创建搜客任务”和“任务队列”，不再误认为必须输入官网。
- 密度：通过。第一屏同时承载表单、渠道、任务状态、结果摘要，留白收紧但不拥挤。
- 可读性：通过。表单字段清晰，状态色只用于动作和风险，不用花哨彩条。
- 可操作性：通过。搜索、AI 配置、导出、生成待办、同步商机都在可预期位置。

## 目标

把“Codex 自动搜客”的能力产品化为 CRM 的上游获客模块，形成从搜索条件、平台询盘/B2B目录/公开搜索/官网解析、AI 清洗、人工筛选、同步客户/商机/待办的闭环。

本次先完成设计，不直接改动业务代码。后续开发时以本文件作为验收基准。

## 产品定位

模块名称：自动获客

导航位置：左侧主导航，放在“客户”和“商机”之前或之间。

设计原因：

- 搜客不是小工具，而是外贸销售新客户开拓的主流程入口。
- 现有“小工具”里的官网产品扒取、AI 配置、OCR 名片识别可以作为底层能力被复用。
- 搜客结果必须能进入客户、商机、待办、计划任务，否则只是数据列表，无法形成销售动作。

## 参考目标

参考 Clay、Apollo、Hunter、ZoomInfo、Clearbit/HubSpot Enrichment、Pipedrive Prospector 一类产品的共同模式：

- 先定义 ICP 搜索策略。
- 多数据源并行找公司。
- 对公司、官网、联系人、联系方式做清洗补全。
- 给结果打分、去重、标注来源。
- 人工确认后批量进入 CRM。
- 后续触达动作沉淀为待办、商机和跟进提醒。

## 重新定稿核心流程

1. 输入获客条件
   - 产品：LED 灯具、五金工具、家居用品等。
   - 国家/地区：德国、英国、土耳其、阿联酋等。
   - 行业/场景：水处理、石油化工、锅炉、泵阀、自动化系统集成。
   - 客户类型：经销商、系统集成商、OEM、EPC、MRO、终端工厂。
   - 询盘/平台来源：Alibaba RFQ、Made-in-China Easy Sourcing、Global Sources RFQ、Europages、Google/Web、官网列表。
   - 排除词：招聘、二手、维修、学校、新闻、目录垃圾页。

2. 系统生成搜索入口与解析任务
   - 平台询盘源生成对应搜索入口，不把不可控平台结果伪装成已抓取数据。
   - 官网/域名源进入真实解析接口，生成候选客户。
   - AI 配置后，进行字段提取、ICP 评分、下一步建议。

3. 结果必须服务业务
   - 每条结果至少要能做四件事之一：同步商机、生成待办、导出、打开来源继续核实。
   - 不能只是静态展示“来源很多”；没有接入的来源只作为搜索入口或预留，不进入虚假结果。

4. 推到商机
   - 勾选结果后同步客户和询盘商机。
   - 商机带入产品、来源、国家、说明和下一步动作。

## 三轮自我辩论：重做版

### 第一轮：做“平台型大屏”还是“搜索工具”？

观点 A：做平台型大屏，看起来完整，有任务卡、流水线、数据源区。

问题：销售真正要的是搜索和结果，静态区域越多越像演示稿，不能服务获客。

结论：砍掉大面积静态说明。页面只保留搜索条件、结果、结果动作、平台搜索入口。

### 第二轮：是否伪造大平台询盘结果？

观点 A：可以把 Alibaba/Made-in-China/Global Sources 结果模拟成候选客户，界面更丰满。

问题：这会污染 CRM，用户以为真的抓到了询盘客户，实际不可用。

结论：不伪造。未接入 API/爬取适配器的平台只生成可点击搜索入口；真实结果来自官网解析、用户导入、后续 API 适配器。

### 第三轮：搜索条件应该宽还是细？

观点 A：只要一个搜索框，像搜索引擎。

观点 B：外贸获客需要产品、国家、行业、客户类型、平台来源、排除词，否则结果太散。

结论：采用“一个主搜索框 + 关键过滤条件”。搜索要快，但过滤条件必须存在，尤其是行业、国家、客户类型和询盘平台。

### 第四轮：结果列表是展示信息还是服务动作？

观点 A：结果列表展示字段即可。

问题：展示字段不能推动销售，仍然要手工搬运。

结论：结果列表必须支持勾选、编辑、同步商机、生成待办、导出。右侧只显示当前结果画像和动作，不再堆平台说明。

1. 新建搜客任务
   - 输入产品关键词，例如 `LED work light`、`cordless drill`、`home storage`。
   - 选择国家/地区，例如德国、英国、土耳其、印度、阿联酋。
   - 选择客户类型：经销商、系统集成商、OEM、EPC、MRO、终端工厂、贸易商。
   - 输入排除词，例如招聘、二手、维修、学校、新闻。
   - 选择数据源和结果数量。

2. 获取候选结果
   - 免费源先跑：官网搜索、公开公司注册数据、LEI、Wikidata、Common Crawl、手工 URL。
   - 付费源预留：Apollo、Hunter、People Data Labs、Crunchbase、BuiltWith、ImportGenius 等。
   - 已有“官网产品扒取”能力作为手工 URL/域名解析器接入。

3. AI 解析与规则兜底
   - 未配置 AI：使用规则解析，提取标题、官网、国家、描述、邮箱/电话等公开信息。
   - 已配置 AI：AI 负责行业判断、产品匹配、字段抽取、ICP 打分、去重建议、触达理由。
   - AI 不自动同步客户，必须经过人工勾选确认。

4. 人工筛选
   - 结果表支持勾选、批量编辑、批量忽略、批量同步。
   - 每条记录展示来源、置信度、ICP 分、重复状态、建议动作。
   - 右侧详情抽屉显示官网摘要、匹配原因、联系人线索、原始来源链接。

5. 同步到系统
   - 同步到线索/客户。
   - 创建商机，带入产品、国家、来源、说明。
   - 创建待办，例如“核实联系人”“发送首次开发信”“补齐 WhatsApp”。
   - 推到计划任务，例如“德国经销商首轮筛选 30 家”。
   - 支持导出 Excel/CSV。

- 顶部：策略摘要、数据源状态、耗时、结果数量。
- 中部：紧凑表格。
- 右侧：详情抽屉。
- 底部：批量动作栏，只有勾选结果后出现。

表格字段：

- 勾选
- 公司名
- 国家
- 业务类型
- 官网
- 联系人
- 联系方式
- 产品匹配
- ICP 分
- 置信度
- 来源
- 重复状态
- 状态

### AI 模型配置入口

入口位置：

- 搜客页右上角“AI 配置”。
- 系统设置中保留同一配置入口。

配置字段复用现有 AI 配置：

- 服务商名称
- Base URL
- API Key
- 模型名
- 温度
- 最大输入长度
- 测试按钮

未配置 AI 时：

- 搜客可运行。
- AI 解析、智能评分、触达理由、自动字段补全显示锁定状态。
- 提示用户“配置 OpenAI 兼容模型后启用”。

## 与现有模块联动

### 客户

- 搜客结果同步时先按 `官网域名 + 公司名 + 国家` 查重。
- 查到已有客户时，不新建；允许把搜客信息追加到客户备注或来源记录。
- 客户详情右侧增加“来源搜客任务”和“相关搜客记录”。

### 商机

- 同步时可创建商机。
- 商机产品字段带入关键词或 AI 识别产品。
- 商机来源写入“智能搜客 / 数据源名称”。
- 若客户已存在，商机关联现有客户。

### 待办

- 勾选结果后批量生成待办：
  - 核实联系人
  - 查找采购/工程负责人
  - 发送首次开发信
  - 3 天后跟进
  - 补齐 WhatsApp/微信
- 默认归属当前用户，不给主管实时看待办内容，延续既有隐私边界。

### 计划任务

- 计划任务页增加“从搜客策略生成任务”。
- 例如：`完成德国经销商搜索 50 家`、`筛掉重复客户`、`同步 15 个高分商机`。

### 跟进提醒

- 搜客同步商机时，可自动生成首次跟进提醒规则。
- 未同步的候选结果不进入提醒，避免提醒噪音。

### 导入导出

- 搜客结果支持 Excel/CSV 导入导出。
- 导出字段与用户要求一致：公司名、业务、国家、官网、联系人、联系方式、说明。
- 额外增加来源、置信度、ICP 分、同步状态。

## 权限与数据隔离

- 普通销售：只能看到自己的搜客任务和结果。
- 主管：可以查看团队搜客任务统计和同步到客户/商机后的业务数据，但不直接查看员工个人待办。
- 管理员/超级管理员：可以管理数据源配置、查看全部搜客任务、维护 API Key。
- AI 配置：默认个人配置；管理员可设置团队默认模型。
- 收费 API Key：默认只有管理员/超级管理员可配置。

## 同步策略

默认不自动同步，采用“人工审核后同步”。

同步前校验：

- 公司名不能为空。
- 官网或联系方式至少有一个。
- 重复客户要提示。
- 低置信度结果默认不勾选。

同步动作：

- 新建客户或关联已有客户。
- 可选新建商机。
- 可选创建待办。
- 可选推入计划任务。

## 三轮自我辩论

### 第一轮：放在小工具还是独立模块？

观点 A：放在小工具，开发成本低，能复用现有官网扒取界面。

观点 B：独立模块，因为搜客是外贸销售获客的主流程，不是一次性工具。

结论：独立模块。小工具里的官网扒取和 AI 配置作为底层能力复用，左侧主导航新增“搜客”。

### 第二轮：AI 是否必须配置后才能用？

观点 A：必须配置 AI，否则结果质量低。

观点 B：AI 不是必须。很多基础搜索、注册数据、官网抓取可以先用规则跑，AI 只增强解析和打分。

结论：AI 可选。未配置时可用规则模式；配置后开启 AI 解析、ICP 打分、触达理由和字段补全。

### 第三轮：是否自动同步到客户/商机？

观点 A：自动同步效率高，像自动获客机器人。

观点 B：外贸客户数据一旦乱进 CRM，后续会污染客户池、报表和跟进提醒。

结论：不自动同步。必须人工勾选确认；批量同步时做去重、低置信度拦截和字段校验。

### 第四轮：免费 API 是否要一次性全接？

观点 A：全部接入显得完整。

观点 B：很多免费源地域有限、字段不全或成本在解析端；一次全接会让系统复杂且不稳定。

结论：首期接入“手工 URL/官网解析 + GLEIF + Companies House + 可配置搜索 API”。OpenCorporates、Common Crawl、SEC、VIES、Wikidata 做二期或辅助验证。

### 第五轮：结果列表做轻还是做重？

观点 A：轻列表，只显示要求字段。

观点 B：优秀搜客工具核心价值在筛选和决策，必须显示分数、来源、重复状态、置信度。

结论：列表保持紧凑但信息密度高。基础字段满足导出，增强字段用于判断质量。

## 最终定稿

首期开发范围：

1. 新增“智能搜客”主导航和页面。
2. 新增搜客任务、搜客结果、数据源配置的数据模型和 MySQL 表。
3. 复用现有 AI 配置，新增搜客页入口。
4. 复用现有官网解析能力，扩展为批量候选结果。
5. 接入 GLEIF 和 Companies House 两个可落地免费数据源。
6. 预留 Brave/SearchAPI/SerpApi/Hunter/Apollo/PDL 等配置位，不默认启用。
7. 结果表支持编辑、勾选、批量忽略、批量同步、导出。
8. 同步到客户、商机、待办、计划任务。
9. 权限按当前用户隔离，管理员可看全局配置和统计。
10. 补后端自检与 Playwright 页面级自测。

首期不做：

- 不做未经确认的自动外呼/自动邮件发送。
- 不做绕过目标网站规则的大规模爬虫。
- 不做联系人隐私数据的强行抓取。
- 不做所有收费 API 的真实接入，只预留配置和适配层。

## 验收口径

- 普通用户登录后，只看到自己的搜客任务和结果。
- 未配置 AI 时，可以用规则模式完成一次搜客并生成候选结果。
- 配置 AI 后，结果出现 AI 解析、ICP 分和触达理由。
- 勾选候选结果后，可以同步到客户和商机。
- 同步时能识别已有客户，避免重复创建。
- 可以从候选结果创建待办。
- 可以从候选结果推到计划任务。
- 搜客结果可以导出 Excel/CSV。
- 刷新页面后任务、结果和配置仍然存在。
- Playwright 能覆盖新建任务、查看结果、编辑结果、批量同步、权限隔离。
