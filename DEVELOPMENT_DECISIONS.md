# CRM 开发功能辩论与阶段计划

## 架构辩论

| 议题 | 方案 A | 方案 B | 结论 |
|---|---|---|---|
| 前端框架 | React + Vite + TypeScript，组件化快 | Vue + Element Plus，后台组件成熟 | 当前从静态原型迁移，React 更适合快速组件化和测试，先用 React |
| 后端 | Node + Express 快速交付 | Java Spring Boot 更企业级 | MVP 先用 Node + Express，接口设计保持 REST，后续可替换为 Java |
| 数据库 | SQLite 开发态零配置 | MySQL 符合最终要求 | 开发态 SQLite，模型按 MySQL 设计，README 保留 MySQL 部署目标 |
| 权限 | 前端模拟 | 后端 JWT + 数据范围过滤 | 必须后端控制。前端只负责展示，后端按角色过滤数据 |
| 报表导出 | 首轮做页面汇报 | 首轮做真实 PPT/PDF 导出 | 首轮先做 PPT 级页面，二轮接入 PDF/PPT 导出 |

## MVP 模块优先级

1. 登录与账号管理：所有数据权限的入口，必须先做。
2. 工作台与待办清单：用户每天打开系统的第一工作面。
3. 客户与线索：CRM 核心对象，OCR 同步也依赖它。
4. 资料维护与在线考试：外贸产品知识沉淀和销售能力管理。
5. 小工具 OCR：展会名片录入效率高，适合形成差异化。
6. 报表汇报页：先实现前端汇报 Deck，后续做导出。

## 权限规则

- 业务员：仅可查看本人客户、本人线索、本人待办、本人考试。
- 主管：可查看本团队全部客户、线索、待办、考试成绩，可分配任务。
- 管理员：全量账号、权限、配置、审计。

## 本轮开发目标

- 前后端分离项目可启动。
- 后端提供登录、账号、客户、待办、资料、考试、OCR、报表摘要接口。
- 前端实现登录、权限切换、工作台、客户、资料、考试、小工具、报表、账号管理页面。
- 至少完成接口自测和前端构建验证。
